# Interaction Model

## Основная идея

Пользователь взаимодействует с Нейтом как с персональным советником, а не как с формой ввода задач. Интерфейсы могут быть разными, но модель одна: пользователь сообщает контекст, Нейт сохраняет, уточняет, планирует и помогает принимать решения.

## Web Chat

Web chat — основной будущий интерфейс Нейта.

Назначение:

- полноценный диалог;
- планирование дня и недели;
- стратегические сессии;
- пересмотр целей;
- объяснение рекомендаций;
- уточнение контекста.

Web chat должен уметь показывать не только ответ, но и какие данные Нейт использовал: активные цели, задачи, решения, последние дневниковые сигналы.

## Dashboard

Dashboard — панель состояния, а не основной разговорный интерфейс.

Он должен показывать:

- текущий фокус;
- задачи;
- дедлайны;
- состояние энергии;
- активные решения;
- проекты;
- недельный план;
- предупреждения о перегрузе.

На текущем этапе dashboard не реализуется.

## Telegram

Telegram — дополнительный быстрый канал для capture.

Назначение:

- быстро записать мысль;
- добавить задачу;
- зафиксировать состояние;
- отправить короткий дневниковый лог;
- получить краткий ответ.

Telegram не должен быть главным местом стратегического планирования. На текущем этапе Telegram не реализуется.

## Основные сценарии

### Быстрый capture

```text
Добавь задачу: завтра 40 минут Speakout.
```

Ожидаемое поведение:

- сохранить raw capture;
- классифицировать как task;
- создать задачу;
- вернуть краткое подтверждение.

### Запрос на план

```text
Давай на неделю сделаем акцент на спорте, но ВКР не бросаем.
```

Ожидаемое поведение:

- классифицировать как planning_request;
- создать запись в decisions;
- предложить план или уточнить ограничения.

### Фиксация решения

```text
Решил временно снизить английский до 20 минут в день.
```

Ожидаемое поведение:

- классифицировать как decision;
- сохранить причину и impact;
- учитывать решение в будущих планах.

### Дневниковый лог

```text
Сегодня усталость 7 из 10, сделал работу, английский не трогал.
```

Ожидаемое поведение:

- классифицировать как daily_log;
- сохранить энергию и факты дня;
- учитывать в планировании.

### Блокер

```text
Я застрял с подключением Telegram webhook.
```

Ожидаемое поведение:

- классифицировать как blocker;
- создать задачу с высоким приоритетом;
- предложить unblock action.

## Команды

Будущие команды:

- "Нейт, составь план на сегодня";
- "Нейт, пересобери неделю";
- "Нейт, что сейчас важнее всего?";
- "Нейт, покажи активные решения";
- "Нейт, почему эта задача в приоритете?";
- "Нейт, снизь нагрузку на завтра";
- "Нейт, сделай weekly review";

## Тон взаимодействия

Нейт должен быть:

- спокойным;
- ясным;
- прямым без давления;
- бережным к энергии пользователя;
- конкретным в следующих шагах;
- честным про неопределенность.
