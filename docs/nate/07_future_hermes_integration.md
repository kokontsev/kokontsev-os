# Future Hermes Integration

## Почему Hermes не используется как ядро сейчас

На текущем этапе Kokontsev OS строит базовый фундамент:

- capture pipeline;
- классификация;
- routing;
- Supabase storage;
- OpenRouter provider;
- документы архитектуры Нейта.

Hermes не используется как ядро сейчас, потому что преждевременное подключение agent runtime усложнит систему раньше, чем стабилизированы данные, типы, правила routing и модель памяти.

Сначала нужно понять, какие задачи Нейт решает как продукт и какие данные ему нужны. Потом можно подключать более сложный runtime.

## Как Hermes можно подключить позже

Hermes может стать дополнительным слоем, когда появятся:

- стабильная модель памяти;
- context pack builder;
- понятные tools;
- permissions model;
- audit log;
- rollback patterns;
- web chat interface.

Подключение должно быть эволюционным: Hermes не заменяет Kokontsev OS, а получает роли внутри нее.

## Возможные роли Hermes

### Memory Layer

Hermes может помогать:

- извлекать факты из raw memory;
- собирать synthesized memory;
- искать похожие ситуации;
- поддерживать context pack;
- помечать устаревшие выводы.

### Skills Layer

Hermes может предоставлять специализированные skills:

- weekly review skill;
- strategy review skill;
- project breakdown skill;
- blocker resolution skill;
- decision analysis skill;
- context cleanup skill.

### Agent Runtime

Hermes может стать runtime для многошаговых задач:

- собрать контекст;
- найти релевантные решения;
- предложить план;
- создать черновик изменений;
- запросить подтверждение;
- применить изменения;
- записать audit log.

## Границы Hermes

Даже если Hermes будет подключен, он не должен:

- обходить permissions model;
- менять стратегию без подтверждения;
- выполнять внешние действия без разрешения;
- становиться единственным источником правды;
- скрывать цепочку значимых действий от audit log.

## Интеграционный принцип

Kokontsev OS остается системой данных и правил. Нейт остается пользовательским помощником и интерфейсом мышления. Hermes может стать исполнительным и когнитивным runtime-слоем, но не должен подменять продуктовую архитектуру.
